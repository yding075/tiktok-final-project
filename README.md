# Hashtag-Engagement-Analysis
Analyzing the Relationship Between Keyword Popularity and Video Engagement on Video Platforms.

# Research Question
Does keyword popularity influence video engagement?

# Project Description
This project analyzes the relationship between keyword popularity and video engagement on video platforms.
The goal is to understand whether more popular topics are associated with higher engagement metrics such as views, likes, and comments.

# Data sources
1. YouTube API — collects keyword-based engagement data such as views, likes, and comments.
2. Kaggle dataset — provides large-scale YouTube video statistics for validation.
3. Google Trends — measures keyword popularity over time.

# Project Structure

- `src/` — source code for data collection and analysis  
- `data/` — collected datasets  
- `results/` — generated visualizations  
- `doc/` — project report  
- `tests.py` — unit tests  
- `requirements.txt` — dependencies
- `main.py` — entry point for running pipelines  

# Outputs
Generated figures are saved in the `results/` folder.

# Results 
-Makeup videos show the highest average views, indicating strong visual-driven engagement.
-Food videos receive the highest average likes.
-Makeup videos generate the most comments.
-Keyword popularity is related to engagement, but the relationship is not always consistent.

# Challenges
-TikTok data was difficult to access because of the lack of a stable public API.
-Google Trends requests were limited by HTTP 429 errors.
-Multiple datasets required format standardization.
-Small-scale API data and large-scale Kaggle data were used differently for analysis and validation.

# Generative AI Disclosure


Generative AI tools (ChatGPT) were used as a support tool during this project.

Specifically, AI was used in the following ways:

1. **Debugging and improving API data collection**  
   While implementing the Google Trends data pipeline, I encountered rate limiting issues (HTTP 429 errors). 
I used ChatGPT to help diagnose the issue and improve the code by adding delays (`time.sleep`) between requests and reducing the number of queries. 
These adjustments made the data collection process more stable and reliable.

2. **Refactoring and simplifying code**  
   Some parts of the code were initially too long or complex. I consulted ChatGPT for suggestions on improving code structure and readability. 
Based on these suggestions, I simplified functions and reorganized logic to make the code more efficient and easier to understand.

3. **Exploring alternative data sources**  
   Initially, I planned to collect data from TikTok, but encountered limitations due to the lack of a stable public API. 
I used ChatGPT to explore alternative data sources, which led to the decision to use the YouTube Data API and Google Trends as more reliable and accessible sources.

All outputs generated by AI were carefully reviewed, modified, and validated by the author before being included in the final project.
